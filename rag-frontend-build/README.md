# rag-frontend-build — static frontend

Part of [FRAGATA](../README.md). This is a pre-built static bundle (HTML + CSS + JS) served by Nginx on the VM. It is **not the source** of the frontend — it is the compiled output of `scripts/build_frontend_bundle.sh` from the `rag/` component.

## Contents

```
rag-frontend-build/
├── index.html       # Entry point; contains the backend API base URL in a <meta> tag
└── static/
    ├── app.js       # All frontend logic
    └── style.css    # Styles
```

The backend URL is embedded in `index.html`:

```html
<meta name="rag-api-base-url" content="http://<VM_IP>:8010" />
```

## Deploying to Nginx

```bash
# Copy files to the Nginx web root
sudo rsync -a --delete rag-frontend-build/ /var/www/rag-frontend/

# Point Nginx at it (config is in rag/deploy/cloud_srv_cesga/nginx/)
sudo cp /opt/rag/deploy/cloud_srv_cesga/nginx/rag-frontend.conf /etc/nginx/sites-available/
sudo ln -sf /etc/nginx/sites-available/rag-frontend.conf /etc/nginx/sites-enabled/
sudo nginx -t && sudo systemctl reload nginx
```

## Rebuilding for a different backend URL

This directory is regenerated by running the build script from within `rag/`:

```bash
cd ../rag
scripts/build_frontend_bundle.sh \
  --api-base-url http://<VM_IP>:8010 \
  --output-dir ../rag-frontend-build
```

Then redeploy as above.

## CORS

For the frontend to reach the API from a different origin, set `RAG_CORS_ALLOW_ORIGINS` in the VM's backend environment file (`rag/deploy/cloud_srv_cesga/env/rag-backend.env.example`) to the frontend URL.
